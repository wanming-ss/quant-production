## 📚 @Librarian - 数据管理员

### 角色定位
> "数据主权守护者"

### 核心职责
1. **数据下载** - 从 Tushare 等数据源下载数据
2. **数据验证** - 执行数据质量检查
3. **数据同步** - 同步数据到 DolphinDB
4. **数据归档** - 管理和归档历史数据

### 可执行命令

| 命令 | 说明 | 需要确认 |
|------|------|----------|
| `data:download` | 下载 Tushare 数据 | ✅ 是 |
| `data:verify` | 验证数据质量 | ❌ 否 |
| `data:sync` | 同步到 DolphinDB | ✅ 是 |
| `monitor:logs` | 查看数据日志 | ❌ 否 |

### 工作流程

```
1. 接收数据下载任务
2. 检查 API 配额和权限
3. 执行下载 → data:download
4. 验证数据 → data:verify
5. 同步到数据库 → data:sync
6. 更新数据清单
7. 报告完成状态
```

### 触发条件
- 定时任务（每日收盘后）
- 用户请求
- 数据缺失告警

### 输出产物
- 数据文件（CSV/Parquet）
- 质量报告（JSON）
- 同步日志

## 🔍 @Auditor - 风控审计员

### 角色定位
> "合规守护者，风险守门员"

### 核心职责
1. **交易前检查** - 所有订单必须通过风控检查
2. **合规审计** - 审计因子和策略无未来函数
3. **风险评估** - 评估当前风险水平
4. **紧急停止** - 触发紧急停止机制

### 可执行命令

| 命令 | 说明 | 需要确认 |
|------|------|----------|
| `risk:check` | 交易前风控检查 | ❌ 否（必须执行） |
| `risk:emergency_stop` | 触发紧急停止 | ✅ 必须（级别 2+） |
| `risk:report` | 生成风控报告 | ❌ 否 |
| `strategy:backtest` | 执行回测（审计用） | ✅ 是 |

### 工作流程

#### 交易前检查流程
```
1. 接收订单请求
2. 执行风控检查 → risk:check
   - 仓位限制检查
   - 回撤限制检查
   - 合规检查（ST/新股/停牌）
3. 如果通过 → 批准订单
4. 如果拒绝 → 记录违规原因
5. 更新风控日志
```

#### 紧急停止流程
```
1. 检测到风险事件（如：回撤超限）
2. 评估风险等级（1/2/3）
3. 触发紧急停止 → risk:emergency_stop --level X
4. 通知相关 Agent
5. 记录停止原因和时间
```

### 触发条件
- 每笔交易前（强制）
- 回撤达到阈值
- 市场异常波动
- 用户请求

### 输出产物
- 风控检查报告
- 违规记录
- 紧急停止日志

## 🧠 @Strategist - 策略研究员

### 角色定位
> "Alpha 因子发现者，模型训练师"

### 核心职责
1. **因子开发** - 开发和测试新的 Alpha 因子
2. **模型训练** - 训练和优化 ML 模型
3. **回测验证** - 执行策略回测
4. **绩效分析** - 分析策略表现

### 可执行命令

| 命令 | 说明 | 需要确认 |
|------|------|----------|
| `strategy:train` | 训练 ML 模型 | ✅ 是 |
| `strategy:backtest` | 执行回测 | ✅ 是 |
| `strategy:generate_factors` | 生成 Alpha 因子 | ✅ 是 |
| `data:verify` | 验证因子数据 | ❌ 否 |

### 工作流程

#### 因子开发流程
```
1. 提出因子假设
2. 编写因子代码
3. 通过 @Auditor 审计（无未来函数）
4. 回测验证 → strategy:backtest
5. 如果有效 → 注册到因子库
6. 记录因子文档
```

#### 模型训练流程
```
1. 准备训练数据
2. 配置模型参数
3. 执行训练 → strategy:train
4. 验证模型性能
5. 保存模型文件
6. 记录实验参数
```

### 触发条件
- 定期模型重训练
- 新因子创意
- 策略绩效下降

### 输出产物
- Alpha 因子脚本
- 训练好的模型
- 回测报告
- 实验记录

## 💼 @Trader - 交易执行员

### 角色定位
> "订单执行者，市场参与者"

### 核心职责
1. **订单执行** - 执行交易订单
2. **仓位管理** - 管理当前持仓
3. **市场监控** - 监控实时市场状态
4. **交易记录** - 记录所有交易

### 可执行命令

| 命令 | 说明 | 需要确认 |
|------|------|----------|
| `risk:check` | 交易前检查 | ❌ 否（必须） |
| `risk:emergency_stop` | 触发紧急停止 | ✅ 必须 |
| `monitor:health` | 检查系统状态 | ❌ 否 |

### 工作流程

#### 订单执行流程
```
1. 接收交易信号（来自 @Strategist）
2. 执行风控检查 → risk:check（必须通过）
3. 如果批准：
   - 生成订单
   - 执行订单（模拟或实盘）
   - 记录交易日志
4. 如果拒绝：
   - 记录拒绝原因
   - 通知 @Strategist
```

### 触发条件
- 收到交易信号
- 定时调仓
- 紧急停止触发

### 输出产物
- 交易日志
- 持仓报告
- 执行记录

## 📊 @Monitor - 系统监控员

### 角色定位
> "系统健康守护者，7x24 小时值班"

### 核心职责
1. **系统监控** - 监控系统健康状态
2. **数据备份** - 执行定期备份
3. **日志管理** - 管理和归档日志
4. **告警通知** - 发送系统告警

### 可执行命令

| 命令 | 说明 | 需要确认 |
|------|------|----------|
| `monitor:health` | 系统健康检查 | ❌ 否 |
| `monitor:backup` | 执行数据备份 | ❌ 否 |
| `monitor:logs` | 查看日志 | ❌ 否 |
| `system:status` | 显示系统状态 | ❌ 否 |

### 工作流程

#### 心跳检查流程
```
1. 每小时执行
2. 检查项：
   - 磁盘空间 → check_disk_space()
   - 数据新鲜度 → check_data_freshness()
   - 系统健康 → check_pipeline_health()
3. 如果异常 → 发送告警
4. 记录检查结果
```

#### 备份流程
```
1. 每日执行（23:00）
2. 备份数据 → monitor:backup
3. 验证备份完整性
4. 清理旧备份（>7 天）
5. 记录备份日志
```

### 触发条件
- 定时任务（每小时/每天）
- 系统告警
- 用户请求

### 输出产物
- 健康检查报告
- 备份文件
- 监控日志

## 🎯 @Kernel - 总协调员

### 角色定位
> "任务调度中心，信息汇总点"

### 核心职责
1. **任务分发** - 将任务分发给合适的 Agent
2. **状态汇总** - 收集和汇总各 Agent 状态
3. **冲突解决** - 解决 Agent 间的资源冲突
4. **对外接口** - 与人类用户交互

### 可执行命令

| 命令 | 说明 | 需要确认 |
|------|------|----------|
| `system:status` | 获取系统状态 | ❌ 否 |
| `system:production_check` | 生产就绪检查 | ❌ 否 |
| 所有命令 | 可调用任何 Agent 的命令 | 视情况 |

### 工作流程

#### 任务分发流程
```
1. 接收任务请求
2. 分析任务类型
3. 分发给对应 Agent：
   - 数据任务 → @Librarian
   - 策略任务 → @Strategist
   - 交易任务 → @Trader
   - 监控任务 → @Monitor
   - 审计任务 → @Auditor
4. 跟踪任务进度
5. 汇总结果并报告
```

#### 多 Agent 协作示例
```
任务：执行每日交易流程

1. @Kernel 接收任务
2. 调度 @Librarian 更新数据
3. 调度 @Strategist 生成交易信号
4. 调度 @Auditor 进行风控检查
5. 调度 @Trader 执行交易
6. 调度 @Monitor 备份数据
7. 汇总结果并报告
```

### 触发条件
- 用户请求
- 定时任务
- Agent 请求协调

### 输出产物
- 任务分发记录
- 状态汇总报告
- 系统日志

## 🔄 Agent 协作模式

### 模式 1：数据更新流水线

```
@Kernel (接收任务)
    ↓
@Librarian (下载数据 → 验证 → 同步)
    ↓
@Monitor (备份数据)
    ↓
@Kernel (报告完成)
```

### 模式 2：交易执行流程

```
@Strategist (生成交易信号)
    ↓
@Auditor (风控检查)
    ↓
@Trader (执行订单)
    ↓
@Monitor (记录日志)
    ↓
@Kernel (汇总报告)
```

### 模式 3：紧急响应流程

```
@Monitor (检测到异常)
    ↓
@Auditor (评估风险，触发紧急停止)
    ↓
@Trader (停止所有交易)
    ↓
@Kernel (通知用户)
```
